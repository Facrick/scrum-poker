# Scrum Poker

Realtime-инструмент для оценки задач командой (Planning Poker) на ~100 участников в комнате.

## Стек

- **Backend:** Java 17 + Spring Boot 3, Spring WebSocket (STOMP over SockJS)
- **Состояние:** in-memory (`ConcurrentHashMap`); на этапе масштабирования — Redis pub/sub
- **Frontend:** статичный HTML/CSS/JS со STOMP-клиентом (без сборщика)

## Запуск

```bash
cd scrum-poker
mvn spring-boot:run
```

Открыть http://localhost:8080 — создать комнату, скопировать ссылку и разослать участникам.

## Возможности (MVP — этап 1–2)

- Создание комнаты и вход по ссылке (без регистрации)
- Роли: модератор (первый вошедший), участник, наблюдатель
- Голосование картами, скрытие значений до вскрытия
- Вскрытие / новый раунд / задание оцениваемой задачи (модератор)
- Удаление участника модератором
- Статистика раунда: среднее, медиана, распределение, индикатор консенсуса
- Авто-переподключение клиента, удаление участника при разрыве соединения
- Лимит 100 участников на комнату

## API

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/rooms` | создать комнату `{name, deck}` |
| GET  | `/api/rooms/{id}` | информация о комнате |
| WS   | `/ws` (SockJS+STOMP) | realtime-канал |

### STOMP destinations
- Клиент → `/app/room/{id}/{join,vote,reveal,reset,story,kick}`
- Сервер → `/topic/room/{id}` (состояние), `/user/queue/me`, `/user/queue/errors`
